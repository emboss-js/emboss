Dependency Arrows

Core

Bottom-to-top SVG bezier paths. Status-based styling: solid (active), dashed (future), faded (done). Arrows flow downward matching reading direction.

Routing Model: Bottom → Top

Arrows exit bottom-center of source bar, enter top-center of target. Always flows downward — no horizontal spaghetti crossing rows.

Exit point
Bottom-center of source bar
Entry point
Top-center of target bar
Direction
Always downward (matches reading order)
Path type
Cubic bezier S-curve
Adjacent rows
Nearly straight vertical drop
Distant rows
Gentle S-curve, wider spread proportional to distance

Status-Based Arrow Styles

Three arrow styles based on dependency status. Creates natural hierarchy: active = clear, future = suggested, done = fades.

Multi-Row S-Curves

Bottom-to-top routing across multiple rows. S-curve width proportional to vertical distance.

Critical Path Highlight

Critical arrows: solid, red, higher opacity. Non-critical fades further. Analyze Bundle

Hover Interaction

12px invisible hit area. Hover brightens arrow and dots. Pairs with dependency indicator dots on bars.

Hit area
12px invisible stroke around path
Hover opacity

1px → 1.5px
Transition

Bar dep dots
Brighten when connected arrow hovered

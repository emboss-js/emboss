Documentation

Everything you need to add a glass-finished Gantt chart to your project. The library isn't built yet — this is the structure the docs will follow.

Installation

Coming soon
// npm
npm install @emboss/core

// yarn
yarn add @emboss/core

// pnpm
pnpm add @emboss/core

Emboss ships as a single package. Core rendering, free extensions, and paid bundles are all exported from @emboss/core. Paid features log a console reminder if used without a license key — they still work, nothing is gated.

Quick Start

Coming soon
import { Emboss } from '@emboss/core'
import { todayMarker, tooltips, arrows } from '@emboss/core/extensions'

const chart = new Emboss('#gantt', tasks, {
  extensions: [todayMarker, tooltips, arrows],
  theme: 'dark',
  view: 'week'
})

That gives you glass-finished bars, a pulsing today marker, hover tooltips, and dependency arrows. Five lines.

The Row Model

Coming soon

Everything in Emboss is a row. Phases, tasks, milestones, subtasks — one data structure, one render pipeline. Extensions enrich rows with additional fields. Renderers consume them.

The Row interface defines the shape of every item in your chart. Full type reference will be documented here once the API is finalized.

State & Mutation

Coming soon

One state object. One source of truth. All changes go through methods on the Emboss instance — setView(), updateRow(), toggleCollapse(). Extensions read state, never write directly.

Views & Density

Coming soon

Four view modes (day, week, month, quarter) and three density options (working, presentation, dense). Each combination produces a specific scale — pixel-per-day, row height, bar height, and label size. Full scale table will be documented here.

Themes

Coming soon

Every visual value flows through --emboss- prefixed CSS custom properties. Swap a class on the container to change the entire feel. Ship with grayscale light and dark. Build your own theme in 20 lines of CSS.

Events

Coming soon

Events are the only way user actions affect state. Drag events, click events, hover events — extensions intercept them, core processes them. Return false from any handler to cancel the action.

Using Extensions

Coming soon

Pass extensions in the config array. Registration order matters — later extensions override earlier ones for the same renderer key. Core registers first, then your extensions in array order.

const chart = new Emboss('#gantt', tasks, {
  extensions: [todayMarker, tooltips, arrows, sidebar, phases]
})

Today Marker

Coming soon

Free extension. Renders a pulsing red line at today's position — line, dot, animated ring, and subtle column glow. Always shows where you are in the timeline. No configuration needed.

Tooltips

Coming soon

Free extension. Cursor-following cards that show task name, dates, progress, and assignee on hover. Boundary-aware positioning so they never clip off-screen. Dark card on light theme, light card on dark theme.

Dependency Arrows

Coming soon

Free extension. Start-to-start right-angle routing between dependent tasks. Hidden by default, visible when you hover a connected bar. Status-based styling — solid lines for active dependencies, dashed for future.

Writing Your Own Extension

Coming soon

Extensions are plain objects with a standard shape. Declare what you provide — renderers, event handlers, row enrichment, CSS — and the core merges them in. One-page guide with a working example.

Sidebar

Organize bundle — coming soon

Full-width or rail mode. Task names with status dots, phase pills with collapse chevrons, assignee avatars. Scroll-synced with the chart. Collapsible to a 48px rail with icon-only view.

Phase Grouping

Organize bundle — coming soon

Collapsible groups that span their children. Color-coded thin bars on the timeline. Click the chevron to expand or collapse. Phase progress is computed from children.

Milestones

Organize bundle — coming soon

Diamond markers for deadlines and checkpoints. Phase-colored borders, progress fill bottom-to-top, glass highlight. 20×20px rotated square with hover scale.

Inline Editing

Organize bundle — coming soon

Click-to-edit on task and phase names directly in the sidebar. Phase-colored border on focus. Enter to save, Escape to cancel.

Emboss Class

Coming soon

The main entry point. Constructor, instance methods, and event subscriptions.

Constructor

new Emboss(selector: string, tasks: Task[], config?: EmbossConfig)

Methods

MethodDescription
setView(view)Switch between 'day', 'week', 'month', 'quarter'. Recalculates scale and re-renders.
setDensity(density)Switch between 'working', 'presentation', 'dense'. Recalculates scale.
setTheme(theme)Apply a theme by swapping the class on the root container.
updateRow(id, changes)Merge partial changes into a row and re-render.
toggleCollapse(id)Expand or collapse a row's children.
on(event, handler)Subscribe to an event.
use(extension)Register an extension at runtime.
destroy()Unmount the chart and clean up event listeners.
